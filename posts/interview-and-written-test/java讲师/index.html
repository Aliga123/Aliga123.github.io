面试前的准备#

一、试讲#

1.问好和开场白#

同学们，大家好！本次由我来给同学上这节课，我的名字叫王力。

2.回顾上次的内容#

我们首先来回顾以下上节课的内容：

  • 基础语法：变量的声明、数据类型、运算符、控制流（if语句、for循环、while循环等）
  • 数据结构：数组（Array）、列表（List）、集合（set）等
  • 函数的创建和调用

3.讲解本次课程的课程目标#

在上课之前，我先问同学们一个问题，同学们是否理解生活中类和对象的概念，大家可以谁便发言没有关系。OK，大家回答的很好。

或许我们经常听到java是一门面向对象语言，其中最重要的思想就是“万物皆对象”。那么什么是面向对象呢？

本节课我们将从以下几个课程目标去了解面向对象：

  • 类的属性和方法
  • 包
  • 对象
  • 构造函数

4.讲解课程内容#

1）类的一般形式

类就是抽象出一群对象的共同表示形式。例如每个人都有名字、年龄、性别等属性，这些属性的集合就称之为一个类，而每个具体的人就是一个对象，他们各种有不同的属性值。

类的定义代码示例如下，其中包括三个属性和一个方法。：

public class Human{
    public String name;
    public String sex;
    public int age;
    
    public void work(){
        System.out.println("i am working");
    }
}

Human tom = new Human();
tom.work()

我们在主函数中根据这个类创建一个对象。

public class test {

	public static void main(String[] args) {
		// TODO Auto-generated method stub
		Human tom = new Human();
		
		tom.work();
		
		tom.age = 3;
		
		System.out.println(tom.age);

	}
}

2）封装

java的三个核心特征：封装、继承、多态

但是我们在使用的时候通常不会直接将类的属性设置为pulic。通过将类的属性设置为私有（‌private）‌，‌只能通过公开的set和get方法进行访问和修改，‌这样可以控制对类内部状态的访问，‌防止外部代码直接修改对象的内部状态，‌从而确保数据的完整性和安全性。‌

public class Human{
    private String name;
    private String sex;
    private int age;
    
	public void setName(String name){
        this.name = name;
    }
    public String getName(){
        return name;
    }
    
    public void setSex(String sex){
        this.sex = sex;
    }
    public String getSex(){
        return sex;
    }
    
    public void setAge(int age){
        this.age = age;
    }
    public int getAge(){
        return age;
    }
}

3）方法的重载

类中同一个方法名，只要传入的参数类型和个数不同，就可以有多个方法实现。

public class Human{
    private String name;
    private String sex;
    private int age;
    
    public void work(){
        System.out.println("i am working");
    }
    
    public void work(int a){
        System.out.println(a);
    }
    
    public void work(int a, int b){
        System.out.println(a+b);
    }
    
    public void work(int a, String s){
        System.out.println("参数string和int被调用");
    }
}

4）构造函数

实际上这个方法就是类自带的方法，我在创建这个类对象的时候就会自动调用该方法，进行对象的初始化操作。

public class Human{
    private String name;
    private String sex;
    private int age;
    
    public Human(String name, String sex, int age) {
    	this.name = name;
    	this.sex = sex;
    	this.age = age;
        
        this.work();
    }
}

构造函数的特点就是函数名和类型相同，且没有返回值。

这就以为我们创建对象的时候还需要传入这三种类型的数据。

没有使用构造函数的时候系统默认是指定了一个空的构造函数。

5.总结#

现在我们来对本节课进行一下总结：

  • 类的属性和方法
  • 对象
  • 方法的重载
  • 构造函数

6.布置课后作业#

最后，给大家布置一个作业，每个人去以生活中的例子去想象一下，哪些东西是对象，从这些对象中提取出一个类出来，来实现本节课的内容。

我的面试、笔试经验#

开始进行自我介绍，和面试官简单的了解了一下。

后面试讲，教学主管和领导试听。教学主管和领导的人很好，每将一部分都会针对我的试讲进行指点不足。我总结了我的不足为一下几点：

  • 语速太快了，没给学生思考的时间
  • 案例没有设计好，只在讲知识，应该融入生活的例子和学生互动。
  • 知识点没讲到位，课程设计不好，这不是一节课能讲的完的。要把东西知识点讲详细，比如javav封装。
  • 还要考虑学生会反问的情况，知识点要学到位，不能让学生尝试疑问，结束每一步为什么这么做。还要做错误示范，不能只做正确示范。

最后，有两个技术面试官进行面试：

  • 首先，自我介绍。
  • 之后就简历上的东西刨根问底，看你对知识点了解的程度。
  • 我说python比较熟练，他就对python开发、数据分析、人工智能（LLM领域）进行详细深入的考察。

我的总结是：简历上写的东西你最好都熟悉。你认为自己熟悉的东西（会刨根问底的），最好把八股文背了。

过了两天后我接到了面试结果通知，主管告诉我一试讲能力不行需要练、技术基础不熟悉不扎实还需要学习，但我还是收到了offer，硕士给了第二档的工资，不高还能接受。我觉得可能是因为公司有培训，所有对能力上的要求可能要求不高，达到合格的程度就行。
